title: Cloudflare Workers
description: Deploy omni-model to Cloudflare Workers and configure it entirely through Worker variables and secrets.
---

Cloudflare Workers run omni-model at the edge. Configuration is read entirely from Worker variables
and secrets; no application configuration file is bundled into the Worker.

<Warning>
Configure a security provider before exposing the Worker. omni-model refuses to start without one,
which prevents an unauthenticated proxy from spending your upstream provider credits.
</Warning>

## Deploy from the repository

[![Deploy to Cloudflare](https://deploy.workers.cloudflare.com/button)](https://deploy.workers.cloudflare.com/?url=https://github.com/tiepvuvan/omni-model)

The button provisions the `OMNI_DO` Durable Object and deploys the complete named-variable starter
from `wrangler.jsonc`. It prompts for the two entries in `.dev.vars.example` during setup:

| Secret | Required | Purpose |
| --- | --- | --- |
| `OMNI_SECURITY_JWT_SECRET` | Yes | HS256 fallback authentication for clients without App Check. |
| `OMNI_PROVIDERS_DEFAULT_API_KEY` | No | Key for the default OpenAI-compatible upstream. Leave blank only for an unauthenticated compatible endpoint. |

The default Worker uses exact Durable Object counters, a JWT verifier, and an OpenAI-compatible
provider pointed at `https://api.openai.com/v1`. It starts with no provider key, so add the key
during setup or later in **Workers & Pages → your Worker → Settings → Variables and Secrets** before
sending model requests. No dashboard configuration is required for the storage binding, rate limits,
or the default provider type and endpoint.

To use Firebase App Check instead, set `OMNI_SECURITY_JWT_ENABLED=false` and add its named profile:

```text
OMNI_SECURITY_FIREBASE_APPCHECK_ENABLED=true
OMNI_SECURITY_FIREBASE_APPCHECK_PROJECT_NUMBER=YOUR_PROJECT_NUMBER
OMNI_SECURITY_FIREBASE_APPCHECK_APP_ID=1:YOUR_PROJECT_NUMBER:ios:YOUR_APP_ID
```

Deploy a new Worker revision after changing a variable or secret.

## Forkless prebuilt Worker

Every release publishes `worker.js` and `wrangler.jsonc`. Download them and deploy without a fork,
clone, or build:

```sh
curl -LO https://github.com/tiepvuvan/omni-model/releases/latest/download/worker.js
curl -LO https://github.com/tiepvuvan/omni-model/releases/latest/download/wrangler.jsonc

npx wrangler deploy \
  --var OMNI_STORAGE_TYPE:durable-object \
  --var OMNI_STORAGE_DURABLE_OBJECT_BINDING:OMNI_DO \
  --var OMNI_SECURITY_JWT_ENABLED:true \
  --var OMNI_SECURITY_JWT_ALGORITHMS:'["HS256"]' \
  --var OMNI_PROVIDERS_DEFAULT_TYPE:openai-compatible \
  --var OMNI_PROVIDERS_DEFAULT_BASE_URL:https://api.openai.com/v1

npx wrangler secret put OMNI_SECURITY_JWT_SECRET
npx wrangler secret put OMNI_PROVIDERS_DEFAULT_API_KEY
```

The uploaded Worker is self-contained. The `OMNI_DO` binding declared in the downloaded
`wrangler.jsonc` provides exact rate-limit and token-budget counters.

## Storage choices

Use Durable Objects for exact shared counters:

```text
OMNI_STORAGE_TYPE=durable-object
OMNI_STORAGE_DURABLE_OBJECT_BINDING=OMNI_DO
```

Use Workers KV when approximate, eventually consistent counters are acceptable:

```text
OMNI_STORAGE_TYPE=cloudflare-kv
OMNI_STORAGE_CLOUDFLARE_KV_BINDING=OMNI_KV
```

Declare `OMNI_KV` in `wrangler.jsonc` before selecting it. App Attest state also requires shared
storage, so use Durable Objects for a typical mobile application.

## Providers and routing

Set provider endpoints and aliases with JSON blocks. This example sends `smart` requests to an
OpenAI-compatible endpoint and all other requests to OpenAI:

```text
OMNI_PROVIDERS_JSON={"openai":{"type":"openai","apiKey":"${OPENAI_API_KEY}"},"fast":{"type":"openai-compatible","baseUrl":"https://api.groq.com/openai/v1","apiKey":"${GROQ_API_KEY}"}}
OMNI_ROUTING_JSON={"routes":[{"name":"smart","when":"request.model == \"smart\"","provider":"fast","model":"llama-3.3-70b-versatile"}],"defaultProvider":"openai"}
```

Store `OPENAI_API_KEY` and `GROQ_API_KEY` as Worker secrets. See the
[configuration reference](/reference/configuration) for all provider, routing, security, and
rate-limit fields.
