The repo ships a `fly.toml` and a `Dockerfile`, so a Fly deploy is `fly launch` plus your provider
secrets. The bundled config runs one scale-to-zero machine with a `/healthz` check on port 8787.

## Deploy

<Steps>
<Step title="Launch the app">
```sh
fly launch --copy-config
```

`--copy-config` creates the app from the committed `fly.toml` (port 8787, `/healthz` health check,
`min_machines_running = 0` so it scales to zero). Run this the first time only.
</Step>

<Step title="Set provider secrets">
```sh
fly secrets set OPENAI_API_KEY=sk-... ANTHROPIC_API_KEY=sk-ant-...
```

Secrets are exposed to `${...}` interpolation in your config as env vars.
</Step>

<Step title="Deploy">
```sh
fly deploy
```
</Step>
</Steps>

## Supplying config

The image ships **no baked config**. It resolves one at startup, first match wins:

1. `OMNI_CONFIG` — inline YAML in an env var.
2. `OMNI_CONFIG_PATH` — path to a YAML file you mounted or baked in.
3. `/app/omni.yaml` — an `omni.yaml` committed at the repo root when the image was built.

<Tabs>
<Tab title="Commit an omni.yaml">
Commit an `omni.yaml` at the repo root before deploying — the `Dockerfile` bakes it into the image
as the default config.
</Tab>
<Tab title="OMNI_CONFIG secret">
Set the whole config as a secret, no file needed:

```sh
fly secrets set OMNI_CONFIG="$(cat my-omni.yaml)"
```
</Tab>
</Tabs>

## Storage

`memory` storage is fine for a **single** machine. If you run more than one machine, counters must
be shared or each machine limits independently — switch storage to `redis` or `postgres`:

```yaml
storage:
  type: redis
  url: ${REDIS_URL}
```

Provision Redis on Fly with `fly redis create` (Upstash), then expose its URL as a secret
(`fly secrets set REDIS_URL=...`).

<Warning>
The starter config has no [security providers](/security/custom-jwt) — add at least one auth verifier
before sharing the URL, or anyone who finds it can spend your API credits.
</Warning>

## Updating

`fly deploy` rebuilds and rolls out. Config-only changes can go through the `OMNI_CONFIG` secret
(`fly secrets set` triggers a restart) with no image rebuild. Rotating a provider key is a
`fly secrets set`.

## Prefer the published image?

You can point Fly at the prebuilt multi-arch image instead of building from source. The same image
runs anywhere Docker does:

```sh
docker run -p 8787:8787 \
  -e OPENAI_API_KEY=sk-... \
  -e OMNI_CONFIG="$(cat omni.yaml)" \
  ghcr.io/tiepvuvan/omni-model:latest
```
