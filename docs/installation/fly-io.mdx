---
title: Fly.io
description: Deploy omni-model to Fly.io as a scale-to-zero container configured entirely with environment variables.
---

The repo ships a `fly.toml` and a `Dockerfile`. Fly configuration and omni-model configuration are
both environment variables; the image does not include an application configuration file.

## Deploy

<Steps>
<Step title="Launch the app">

```sh
fly launch --copy-config
```

`--copy-config` creates the app from the committed `fly.toml` (port 8787, `/healthz`, and
scale-to-zero settings). Run this once.
</Step>

<Step title="Set secrets and configuration">

```sh
fly secrets set \
  OPENAI_API_KEY=sk-... \
  OMNI_JWT_SECRET="$(openssl rand -base64 48)" \
  OMNI_STORAGE_TYPE=memory \
  OMNI_SECURITY_PROVIDERS_JSON='[{"type":"jwt","secret":"${OMNI_JWT_SECRET}","algorithms":["HS256"]}]' \
  OMNI_PROVIDERS_JSON='{"openai":{"type":"openai","apiKey":"${OPENAI_API_KEY}"}}' \
  OMNI_DEFAULT_PROVIDER=openai
```

Fly stores these as service environment variables. omni-model interpolates the two secret references
only inside its own configuration at startup.
</Step>

<Step title="Deploy">

```sh
fly deploy
```
</Step>
</Steps>

## Shared storage

`memory` works for one machine. For multiple machines, use a shared backend. For example, after
provisioning Redis, replace the storage value:

```sh
fly secrets set OMNI_STORAGE_JSON='{"type":"redis","url":"${REDIS_URL}"}' REDIS_URL=redis://...
```

Use `postgres` with `{"type":"postgres","url":"${DATABASE_URL}"}` in the same way.

## Updating

Update a configuration value with `fly secrets set`; Fly restarts the machines with a new revision.
For complex providers or model routing, edit `OMNI_PROVIDERS_JSON` or `OMNI_ROUTING_JSON`. See the
[configuration reference](/reference/configuration) for the complete environment-variable surface.

## Prefer the published image?

Point Fly at a release-pinned GHCR image instead of building from source. The same environment
variables work anywhere Docker runs:

```sh
docker run -p 8787:8787 \
  -e OPENAI_API_KEY=sk-... \
  -e OMNI_JWT_SECRET=replace-with-a-long-random-secret \
  -e OMNI_STORAGE_TYPE=memory \
  -e 'OMNI_SECURITY_PROVIDERS_JSON=[{"type":"jwt","secret":"${OMNI_JWT_SECRET}","algorithms":["HS256"]}]' \
  -e 'OMNI_PROVIDERS_JSON={"openai":{"type":"openai","apiKey":"${OPENAI_API_KEY}"}}' \
  -e OMNI_DEFAULT_PROVIDER=openai \
  ghcr.io/tiepvuvan/omni-model:latest
```
