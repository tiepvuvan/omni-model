title: Configuration reference
description: Configure omni-model entirely with environment variables, including providers, security, rate limits, and model routing.

omni-model reads its configuration entirely from environment variables. It does not read YAML files,
mounted configuration files, or an inline YAML variable. Every configuration change is a platform
environment change and creates a new deployment revision on platforms such as Cloud Run.

<Info>
Keep credentials in your platform's secret store. Configuration values can reference them with
`${VAR}` interpolation, so a provider block can use `${OPENAI_API_KEY}` without copying the key
into a JSON configuration value.
</Info>

## Choose the right environment shape

Use the smallest shape that fits the change:

| Use case | Environment variable | Example |
| --- | --- | --- |
| Complete configuration | `OMNI_CONFIG_JSON` | A JSON object containing all top-level blocks. |
| Complex provider set | `OMNI_PROVIDERS_JSON` | A JSON object keyed by provider ID. |
| Routing and model aliases | `OMNI_ROUTING_JSON` | A JSON object with routes, model rules, and a default. |
| Rate-limit rules | `OMNI_RATE_LIMITS_JSON` | A JSON array of rules. |
| Security providers | `OMNI_SECURITY_JSON` or `OMNI_SECURITY_PROVIDERS_JSON` | A JSON object or array. |
| One default provider | `OMNI_PROVIDERS_DEFAULT_*` | `OMNI_PROVIDERS_DEFAULT_TYPE=openai`. |
| One built-in verifier | `OMNI_SECURITY_<VERIFIER>_*` | `OMNI_SECURITY_FIREBASE_AUTH_ENABLED=true`. |
| One simple value | `OMNI_STORAGE_TYPE`, `OMNI_ROUTING_DEFAULT_PROVIDER`, `OMNI_SERVER_LOG_LEVEL`, or `OMNI_SECURITY_MODE` | `OMNI_STORAGE_TYPE=firestore`. |
| One deeply nested override | `OMNI__…` | `OMNI__ROUTING__ROUTES__0__MODEL=gpt-4o-mini`. |

The precedence order is: `OMNI_CONFIG_JSON`, named JSON blocks, simple variables, then `OMNI__…`
paths. Later values override earlier values at the same path.

## Quick setup with named variables

Use named variables for the common one-provider deployment. They avoid JSON and are the fastest way
to configure Cloud Run, Docker, Fly.io, or a Worker. The built-in provider created this way always
has the routing ID **`default`**; setting any `OMNI_PROVIDERS_DEFAULT_*` variable automatically sets
`routing.defaultProvider` to `default` unless `OMNI_ROUTING_DEFAULT_PROVIDER` or a later `OMNI__…`
override changes it.

```sh
# Storage
OMNI_STORAGE_TYPE=firestore
OMNI_STORAGE_FIRESTORE_COLLECTION=omni_ratelimits

# The default upstream provider
OMNI_PROVIDERS_DEFAULT_TYPE=openai
OMNI_PROVIDERS_DEFAULT_API_KEY='${OPENAI_API_KEY}'

# One or more enabled authentication methods
OMNI_SECURITY_FIREBASE_AUTH_ENABLED=true
OMNI_SECURITY_FIREBASE_AUTH_PROJECT_ID=my-firebase-project
OMNI_SECURITY_FIREBASE_APPCHECK_ENABLED=true
OMNI_SECURITY_FIREBASE_APPCHECK_PROJECT_NUMBER=1234567890
OMNI_SECURITY_FIREBASE_APPCHECK_APP_ID=1:1234567890:ios:abc123
# Opt in only after clients send a fresh limited-use token per request.
OMNI_SECURITY_FIREBASE_APPCHECK_CONSUME=false

# Optional model alias rules are JSON because they are ordered lists.
OMNI_ROUTING_ROUTES='[{"name":"smart","when":"request.model == \"smart\"","provider":"default","model":"gpt-4o-mini"}]'
```

<Note>
`*_ENABLED` switches accept only `true` or `false`. A verifier's companion variables do nothing
until its switch is `true`. Every enabled verifier must validate the same request by default; set
`OMNI_SECURITY_MODE=any` explicitly when either credential may authenticate it.
</Note>

## Client response privacy

The proxy returns only the OpenAI-compatible fields a client needs to use a result: completion or
stream identifiers generated by omni-model, the model alias the client requested, choices, message
content and tool calls, and finish reasons. It never returns upstream usage, token counts, costs,
provider names, service tiers, fingerprints, upstream identifiers, or provider-specific extra
fields. Token usage is retained privately so token-budget rate limits still work.
Provider error text is similarly replaced with a stable proxy error.

The same redaction applies to chat completions (including SSE), embeddings, model ownership, and
Firebase callable responses. The client must not use `usage` for metering; enforce quotas on the
proxy instead.

Use JSON blocks when you need a second named provider, more than one provider of the same type, or
full route/rate-limit objects. The [schema explorer](#environment-schema-explorer) below lists both
the easy variables and advanced fields.

## Full configuration in one variable

`OMNI_CONFIG_JSON` accepts every valid omni-model configuration field. Its value must be a JSON
object. This is useful when your infrastructure manager supports a single JSON secret or variable.

```sh
export OMNI_CONFIG_JSON='{
  "version": 1,
  "server": { "logLevel": "info" },
  "storage": { "type": "memory" },
  "security": {
    "providers": [
      { "type": "jwt", "secret": "${OMNI_JWT_SECRET}", "algorithms": ["HS256"] }
    ]
  },
  "providers": {
    "openai": { "type": "openai", "apiKey": "${OPENAI_API_KEY}" }
  },
  "routing": { "defaultProvider": "openai" }
}'
```

`OPENAI_API_KEY` and `OMNI_JWT_SECRET` remain independent environment variables. Bind them from
Secret Manager rather than placing their values inside `OMNI_CONFIG_JSON`.

## Composable JSON blocks

Named blocks make complex configuration easy to edit independently. They merge with each other and
with `OMNI_CONFIG_JSON`.

```sh
export OMNI_STORAGE_JSON='{"type":"firestore","collection":"omni_ratelimits"}'

export OMNI_SECURITY_PROVIDERS_JSON='[
  {"type":"firebase-app-check","appIds":["1:PROJECT_NUMBER:ios:APP_ID"]},
  {"type":"firebase-auth","projectId":"${GOOGLE_CLOUD_PROJECT}"}
]'

export OMNI_PROVIDERS_JSON='{
  "openai": {"type":"openai","apiKey":"${OPENAI_API_KEY}"},
  "groq": {
    "type":"openai-compatible",
    "baseUrl":"https://api.groq.com/openai/v1",
    "apiKey":"${GROQ_API_KEY}"
  }
}'
```

Available block variables are:

| Variable | Configuration path | JSON type |
| --- | --- | --- |
| `OMNI_CONFIG_JSON` | whole document | object |
| `OMNI_SERVER_JSON` | `server` | object |
| `OMNI_STORAGE_JSON` | `storage` | object |
| `OMNI_SECURITY_JSON` | `security` | object |
| `OMNI_SECURITY_PROVIDERS_JSON` | `security.providers` | array |
| `OMNI_RATE_LIMITS_JSON` | `rateLimits` | array |
| `OMNI_PROVIDERS_JSON` | `providers` | object |
| `OMNI_ROUTING_JSON` | `routing` | object |

## Granular `OMNI__…` paths

Use a double underscore to traverse configuration objects. Uppercase snake case converts to the
lower-camel-case property name. A numeric segment creates an array item.

```sh
# server.maxBodyBytes = 3000000
OMNI__SERVER__MAX_BODY_BYTES=3000000

# security.providers[0].type = "jwt"
OMNI__SECURITY__PROVIDERS__0__TYPE=jwt

# providers.openai.apiKey = "${OPENAI_API_KEY}"
OMNI__PROVIDERS__OPENAI__API_KEY='${OPENAI_API_KEY}'

# routing.routes[0].when = 'request.model == "smart"'
OMNI__ROUTING__ROUTES__0__WHEN='request.model == "smart"'
```

Values beginning with `{`, `[`, or `"` are parsed as JSON. Bare JSON numbers, `true`, `false`, and
`null` become their matching types. Other values remain strings. Use a JSON string such as `"123"`
when a string could otherwise be mistaken for a number.

For provider IDs or third-party option names that do not fit a shell-friendly path, set that object
as JSON instead:

```sh
OMNI_PROVIDERS_JSON='{
  "azure_openai-production": {
    "type": "openai-compatible",
    "baseUrl": "https://example.openai.azure.com/openai/v1",
    "apiKey": "${AZURE_OPENAI_API_KEY}"
  }
}'
OMNI_DEFAULT_PROVIDER=azure_openai-production
```

## Server

Use `OMNI_SERVER_JSON` for several values, or the individual aliases below.

| Path | Variable | Type | Default |
| --- | --- | --- | --- |
| `server.logLevel` | `OMNI_SERVER_LOG_LEVEL` (or legacy `OMNI_LOG_LEVEL`) | `debug` \| `info` \| `warn` \| `error` \| `silent` | `info` |
| `server.trustProxyHeaders` | `OMNI_SERVER_TRUST_PROXY_HEADERS` | boolean | `false` |
| `server.maxBodyBytes` | `OMNI_SERVER_MAX_BODY_BYTES` | positive integer | `131072` (128 KiB) |
| `server.cors` | `OMNI_SERVER_CORS` | JSON object | none |

Example browser CORS policy:

```sh
OMNI_SERVER_JSON='{
  "logLevel":"info",
  "cors":{
    "allowOrigins":["https://app.example.com"],
    "credentials":true,
    "maxAge":3600
  }
}'
```

Requests are capped at **128 KiB** by default, measured in UTF-8 bytes. The proxy checks both
`Content-Length` and streaming body bytes, returning `413 payload_too_large` without buffering an
oversized chunked request. Raise the limit deliberately for large multimodal or embedding inputs:

```sh
OMNI_SERVER_MAX_BODY_BYTES=524288
```

Set `trustProxyHeaders` only behind a reverse proxy or CDN that overwrites forwarding headers.

## Storage

`storage` controls rate-limit counters, token budgets, and Apple attestation state. Set
`OMNI_STORAGE_TYPE` first, then set only the option variables for that type. An option for the wrong
type is rejected during startup validation.

| Storage type | Setup variables | Platform / behavior |
| --- | --- | --- |
| `memory` | `OMNI_STORAGE_TYPE=memory` | One process or local development. Not shared across Cloud Run instances. |
| `firestore` | `OMNI_STORAGE_TYPE=firestore`<br/>`OMNI_STORAGE_FIRESTORE_COLLECTION=omni_ratelimits` | Cloud Run / GCP. Exact shared counters using Application Default Credentials. `collection` is optional. |
| `redis` | `OMNI_STORAGE_TYPE=redis`<br/>`OMNI_STORAGE_REDIS_URL=${REDIS_URL}`<br/>`OMNI_STORAGE_REDIS_KEY_PREFIX=omni:` | Shared exact counters. `keyPrefix` is optional. |
| `postgres` | `OMNI_STORAGE_TYPE=postgres`<br/>`OMNI_STORAGE_POSTGRES_URL=${DATABASE_URL}`<br/>`OMNI_STORAGE_POSTGRES_TABLE=omni_kv`<br/>`OMNI_STORAGE_POSTGRES_MIGRATE=true` | Shared exact counters. `table` and `migrate` are optional. |
| `cloudflare-kv` | `OMNI_STORAGE_TYPE=cloudflare-kv`<br/>`OMNI_STORAGE_CLOUDFLARE_KV_BINDING=OMNI_KV` | Workers only. Counters are eventually consistent. |
| `durable-object` | `OMNI_STORAGE_TYPE=durable-object`<br/>`OMNI_STORAGE_DURABLE_OBJECT_BINDING=OMNI_DO` | Workers only. Exact counters; preferred for App Attest. |

`memory` is the default only when storage is omitted. Use `OMNI_STORAGE_JSON` for third-party
storage options not listed here.

## Security

Every HTTP proxy deployment needs at least one security provider. Every configured verifier must
accept by default; set `OMNI_SECURITY_MODE=any` explicitly to accept the first recognized valid
credential instead.

### Built-in verifier profiles

Set the profile's `*_ENABLED=true` switch, then provide its required fields. You may enable several
profiles; `OMNI_SECURITY_MODE` determines whether any or all must validate. Use
`OMNI_SECURITY_PUBLIC_PATHS='["/v1/models"]'` for paths that bypass authentication.

| Verifier | Enable and required configuration | Optional configuration |
| --- | --- | --- |
| Firebase Auth | `OMNI_SECURITY_FIREBASE_AUTH_ENABLED=true`<br/>`OMNI_SECURITY_FIREBASE_AUTH_PROJECT_ID=my-project` | `OMNI_SECURITY_FIREBASE_AUTH_HEADER`<br/>`OMNI_SECURITY_FIREBASE_AUTH_CLOCK_TOLERANCE_SECONDS` |
| Firebase App Check | `OMNI_SECURITY_FIREBASE_APPCHECK_ENABLED=true`<br/>`OMNI_SECURITY_FIREBASE_APPCHECK_PROJECT_NUMBER=1234567890` outside GCP | `OMNI_SECURITY_FIREBASE_APPCHECK_APP_ID` (one)<br/>`OMNI_SECURITY_FIREBASE_APPCHECK_APP_IDS` (JSON array)<br/>`OMNI_SECURITY_FIREBASE_APPCHECK_HEADER`<br/>`OMNI_SECURITY_FIREBASE_APPCHECK_CONSUME` (default `false`)<br/>`OMNI_SECURITY_FIREBASE_APPCHECK_CLOCK_TOLERANCE_SECONDS`<br/><br/>On Cloud Run/GCE, omit project number to discover it at startup. |
| JWT | `OMNI_SECURITY_JWT_ENABLED=true` plus exactly one of `OMNI_SECURITY_JWT_SECRET`, `OMNI_SECURITY_JWT_JWKS_URL`, or `OMNI_SECURITY_JWT_PUBLIC_KEY` | `OMNI_SECURITY_JWT_ALGORITHMS` (JSON array)<br/>`OMNI_SECURITY_JWT_ISSUER`<br/>`OMNI_SECURITY_JWT_AUDIENCE` (string or JSON array)<br/>`OMNI_SECURITY_JWT_HEADER`<br/>`OMNI_SECURITY_JWT_SCHEME` (`bearer` or `none`)<br/>`OMNI_SECURITY_JWT_USER_ID_CLAIM`<br/>`OMNI_SECURITY_JWT_DEVICE_ID_CLAIM`<br/>`OMNI_SECURITY_JWT_CLOCK_TOLERANCE_SECONDS` |
| Supabase | `OMNI_SECURITY_SUPABASE_ENABLED=true` plus one of `OMNI_SECURITY_SUPABASE_URL`, `OMNI_SECURITY_SUPABASE_JWT_SECRET`, or `OMNI_SECURITY_SUPABASE_JWKS_URL` | `OMNI_SECURITY_SUPABASE_ISSUER`<br/>`OMNI_SECURITY_SUPABASE_AUDIENCE` (string or JSON array)<br/>`OMNI_SECURITY_SUPABASE_HEADER`<br/>`OMNI_SECURITY_SUPABASE_CLOCK_TOLERANCE_SECONDS` |
| Apple App Attest | `OMNI_SECURITY_APP_ATTEST_ENABLED=true`<br/>`OMNI_SECURITY_APP_ATTEST_TEAM_ID`<br/>`OMNI_SECURITY_APP_ATTEST_BUNDLE_ID` | `OMNI_SECURITY_APP_ATTEST_ENVIRONMENT`<br/>`OMNI_SECURITY_APP_ATTEST_CHALLENGE_TTL`<br/>`OMNI_SECURITY_APP_ATTEST_ROOT_CA_PEM`<br/>`OMNI_SECURITY_APP_ATTEST_KEY_ID_HEADER`<br/>`OMNI_SECURITY_APP_ATTEST_ASSERTION_HEADER`<br/>`OMNI_SECURITY_APP_ATTEST_CHALLENGE_HEADER` |
| Apple DeviceCheck | `OMNI_SECURITY_DEVICE_CHECK_ENABLED=true`<br/>`OMNI_SECURITY_DEVICE_CHECK_TEAM_ID`<br/>`OMNI_SECURITY_DEVICE_CHECK_KEY_ID`<br/>`OMNI_SECURITY_DEVICE_CHECK_PRIVATE_KEY` | `OMNI_SECURITY_DEVICE_CHECK_DEVELOPMENT`<br/>`OMNI_SECURITY_DEVICE_CHECK_HEADER`<br/>`OMNI_SECURITY_DEVICE_CHECK_CACHE_TTL` |

Array and object variables use JSON: for example,
`OMNI_SECURITY_JWT_ALGORITHMS='["HS256"]'` and
`OMNI_SECURITY_FIREBASE_APPCHECK_APP_IDS='["1:123:ios:abc","1:123:android:def"]'`.
See the dedicated [security guides](/security/custom-jwt) for request headers and verification
behavior.

```sh
OMNI_SECURITY_JSON='{
  "mode":"all",
  "publicPaths":["/healthz"],
  "providers":[
    {"type":"firebase-app-check","appIds":["1:PROJECT_NUMBER:ios:APP_ID"]},
    {"type":"firebase-auth","projectId":"${GOOGLE_CLOUD_PROJECT}"}
  ]
}'
```

Supported provider types include `jwt`, `firebase-auth`, `firebase-app-check`, `supabase`,
`apple-app-attest`, and `apple-device-check`. Their fields are supplied directly in the JSON entry.
See the dedicated [security guides](/security/custom-jwt) for verifier-specific options and client
headers.

## Providers, endpoints, and model types

### Default provider profile

For one upstream, use `OMNI_PROVIDERS_DEFAULT_*`; it creates a provider named `default` and routes
unmatched requests there automatically. `OMNI_PROVIDERS_DEFAULT_TYPE` is one of `openai`,
`openai-compatible`, `anthropic`, or `google`.

| Variable | Applies to | Type / behavior |
| --- | --- | --- |
| `OMNI_PROVIDERS_DEFAULT_TYPE` | all | Required. Provider implementation type. |
| `OMNI_PROVIDERS_DEFAULT_API_KEY` | OpenAI, Anthropic, Google; optional for compatible | String or `${SECRET}` reference. |
| `OMNI_PROVIDERS_DEFAULT_BASE_URL` | all | URL. Required for `openai-compatible`; optional otherwise. |
| `OMNI_PROVIDERS_DEFAULT_HEADERS` | OpenAI / compatible | JSON object of extra upstream headers. |
| `OMNI_PROVIDERS_DEFAULT_MODELS` | OpenAI / compatible | JSON array used as a fallback `/v1/models` list. |
| `OMNI_PROVIDERS_DEFAULT_ORGANIZATION` | OpenAI / compatible | Optional OpenAI organization header. |
| `OMNI_PROVIDERS_DEFAULT_INCLUDE_STREAM_USAGE` | OpenAI / compatible | Boolean, default `true`. |
| `OMNI_PROVIDERS_DEFAULT_ANTHROPIC_VERSION` | Anthropic | String, default `2023-06-01`. |
| `OMNI_PROVIDERS_DEFAULT_ANTHROPIC_MAX_TOKENS_DEFAULT` | Anthropic | Positive integer, default `4096`. |

```sh
# OpenAI
OMNI_PROVIDERS_DEFAULT_TYPE=openai
OMNI_PROVIDERS_DEFAULT_API_KEY='${OPENAI_API_KEY}'

# Or an OpenAI-compatible endpoint such as Groq, OpenRouter, Together, or a local gateway.
OMNI_PROVIDERS_DEFAULT_TYPE=openai-compatible
OMNI_PROVIDERS_DEFAULT_BASE_URL=https://api.groq.com/openai/v1
OMNI_PROVIDERS_DEFAULT_API_KEY='${GROQ_API_KEY}'
```

### Multiple named providers

`OMNI_PROVIDERS_JSON` is an object keyed by your provider IDs. Each entry must contain `type`.

| Provider type | Required fields | Optional fields |
| --- | --- | --- |
| `openai` | `apiKey` | `baseUrl`, `organization`, `headers`, `models`, `includeStreamUsage` |
| `openai-compatible` | `baseUrl` | `apiKey`, `organization`, `headers`, `models`, `includeStreamUsage` |
| `anthropic` | `apiKey` | `baseUrl`, `version`, `maxTokensDefault` |
| `google` | `apiKey` | `baseUrl` |

Set an endpoint per provider, not globally. This lets one route use OpenAI while another uses an
OpenAI-compatible gateway:

```sh
OMNI_PROVIDERS_JSON='{
  "openai": {"type":"openai","apiKey":"${OPENAI_API_KEY}"},
  "anthropic": {"type":"anthropic","apiKey":"${ANTHROPIC_API_KEY}"},
  "fast": {
    "type":"openai-compatible",
    "baseUrl":"https://api.groq.com/openai/v1",
    "apiKey":"${GROQ_API_KEY}"
  }
}'
```

Provider IDs are your stable routing names. Client model names do not have to match upstream model
names; routes and model rules can replace them.

## Model routing

Set `OMNI_ROUTING_JSON` to configure client-facing aliases, conditional routes, model rules, and a
fallback provider. Routes run in order; the first `when` expression that evaluates to true wins.

For the common fields, these aliases avoid wrapping one value in an object:

| Variable | JSON type | Maps to |
| --- | --- | --- |
| `OMNI_ROUTING_DEFAULT_PROVIDER` | string | `routing.defaultProvider` |
| `OMNI_ROUTING_ALLOWED_MODELS` | string array | `routing.allowedModels` |
| `OMNI_ROUTING_ROUTES` | array | `routing.routes` |
| `OMNI_ROUTING_MODEL_RULES` | array | `routing.modelRules` |

`OMNI_DEFAULT_PROVIDER` remains supported as a legacy alias. Use the `OMNI_ROUTING_*` names for new
deployments. Ordered routes and model rules are arrays, so use JSON for them.

Set an allowlist to restrict clients to exact model names. This applies to chat completions and
embeddings before any upstream call; `/v1/models` then exposes only these client-facing names. An
empty or omitted allowlist permits all model names, preserving the default behavior.

```sh
OMNI_ROUTING_ALLOWED_MODELS='["smart","fast","embed"]'
```

```sh
OMNI_ROUTING_JSON='{
  "routes":[
    {
      "name":"smart-pro",
      "when":"request.model == \"smart\" && has(user.claims.tier) && user.claims.tier == \"pro\"",
      "provider":"anthropic",
      "model":"claude-sonnet-4-5"
    },
    {
      "name":"smart-default",
      "when":"request.model == \"smart\"",
      "provider":"fast",
      "model":"llama-3.3-70b-versatile"
    }
  ],
  "modelRules":[
    {"match":"request.model.startsWith(\"gpt-\")","provider":"openai"},
    {"match":"request.model.startsWith(\"claude-\")","provider":"anthropic"}
  ],
  "defaultProvider":"openai"
}'
```

For a surgical update, override one generated route field:

```sh
OMNI__ROUTING__ROUTES__1__MODEL=llama-3.3-70b-versatile
```

All `when`, `match`, and rate-limit `when` expressions use CEL. Guard optional claims with
`has(user.claims.tier)` before reading them. See [Model routing](/model-routing/routing) for the
available request, user, device, HTTP, and time facts.

## Rate limits

`OMNI_RATE_LIMITS_JSON` (or the shorter alias `OMNI_RATE_LIMITS`) is an ordered JSON array. Every
rule needs a `name`, a key strategy, and at least one `requests` or `tokens` limit. When this
variable is omitted, omni-model limits each authenticated user to **30 requests per hour** and
**30,000 total tokens per day**.

```sh
OMNI_RATE_LIMITS_JSON='[
  {
    "name":"per-user-requests",
    "key":"user",
    "requests":{"limit":30,"window":"1h"}
  },
  {
    "name":"per-user-daily-tokens",
    "key":"user",
    "tokens":{"limit":30000,"window":"1d"}
  }
]'
```

`key` can be `user`, `device`, `ip`, `global`, or `expression`. Use `keyExpression` when the key
is `expression`. Durations are `500ms`, `30s`, `5m`, `1h`, or `1d`.

## Environment schema explorer

This is the canonical configuration object after omni-model merges your environment variables. Use
it as a JSON-schema-style map when an easy variable does not cover an option. Each component validates
its own options at startup, so unsupported fields fail before the service accepts traffic.

| Root field | JSON type | Default | Notes |
| --- | --- | --- | --- |
| `version` | literal `1` | `1` | Compatibility marker. |
| `server` | object | `{ logLevel: "info", trustProxyHeaders: false, maxBodyBytes: 131072 }` | HTTP and logging behavior. |
| `storage` | object | `{ type: "memory" }` | Selects a storage backend and its options. |
| `security` | object | `{ mode: "all", publicPaths: [], providers: [] }` | At least one provider is required for an HTTP proxy. |
| `rateLimits` | array | `[{ per-user-requests: 30/1h }, { per-user-daily-tokens: 30000/1d }]` | Ordered request/token budget rules. |
| `providers` | object keyed by ID | `{}` | Each value has a `type` and type-specific options. |
| `routing` | object | `{ allowedModels: [], routes: [], modelRules: [] }` | Restricts client model names and chooses an upstream. |

### `server` object

| Field | Type | Default |
| --- | --- | --- |
| `logLevel` | `debug` \| `info` \| `warn` \| `error` \| `silent` | `info` |
| `trustProxyHeaders` | boolean | `false` |
| `maxBodyBytes` | positive integer | `131072` (128 KiB) |
| `cors.allowOrigins` | string array | `["*"]` |
| `cors.allowMethods`, `cors.allowHeaders`, `cors.exposeHeaders` | string arrays | omitted |
| `cors.maxAge` | positive integer seconds | omitted |
| `cors.credentials` | boolean | omitted |

### `security` object

| Field | Type | Default | Constraint |
| --- | --- | --- | --- |
| `mode` | `any` \| `all` | `all` | `all` requires every configured verifier; set `any` explicitly to accept either credential. |
| `publicPaths` | string array | `[]` | Exact paths or a trailing `*` prefix. |
| `providers` | array | `[]` | Must contain at least one accepted verifier for `/v1/*`. |

### `rateLimits[]` object

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | string | yes | Stable rule name, shown in rate-limit headers/logs. |
| `when` | CEL string | no | Applies only when it evaluates to `true`. |
| `key` | `user` \| `device` \| `ip` \| `global` \| `expression` | no | Defaults to `user`. |
| `keyExpression` | CEL string | when `key=expression` | Produces the rate-limit key. |
| `requests` | `{ limit: positive integer, window: duration }` | one of requests/tokens | Request-window limit. |
| `tokens` | `{ limit: positive integer, window: duration }` | one of requests/tokens | Prompt + completion token budget. |

### `routing` object

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `allowedModels` | string array | `[]` | Exact client model names allowed for chat and embeddings. When non-empty, `/v1/models` returns this list only. |
| `routes` | array of `{ name, when, provider, model? }` | `[]` | First CEL `when` that is true wins. |
| `modelRules` | array of `{ match, provider, model? }` | `[]` | Fallback matching by client model name. |
| `defaultProvider` | provider ID | omitted | Last fallback; an unmatched request is 404 when omitted. |

See [Model routing](/model-routing/routing) for the CEL request context and a full routing example.

## Validation and interpolation

The service validates the fully merged environment configuration at startup. Unknown top-level
fields, invalid JSON, missing secrets, unknown providers, invalid CEL, or invalid component options
stop the revision before it serves traffic.

String values support:

```text
${NAME}             required environment value
${NAME:-fallback}   environment value with a fallback
$${NAME}            a literal ${NAME}
```

Use `omni-model init` to print a starter set of environment blocks, and
`omni-model deploy` to send those blocks to a supported target.
