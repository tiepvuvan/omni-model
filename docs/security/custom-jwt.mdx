---
title: Custom JWT & JWT-family verifiers
description: Authenticate the self-hosted proxy with any JWT issuer, Firebase Auth, or Supabase using Authorization Bearer tokens.
---

The `jwt`, `firebase-auth`, and `supabase` verifiers all authenticate the
self-hosted proxy by validating a **JSON Web Token** carried in a request
header — by default `Authorization: Bearer <token>`. They share the same JWKS
resolver and error handling; the Firebase and Supabase variants are pre-wired
presets of the generic `jwt` verifier.

These verifiers apply to the **self-hosted HTTP proxy** (configured under
`security.providers`). The serverless [Firebase Extension](/installation/firebase)
uses Firebase Auth natively on callable functions and needs no verifier block.

<Info>
On success, the token subject becomes `user.id` and every claim is exposed to
[routing and rate-limit CEL rules](/model-routing/routing) as `user.claims`.
Accessing a missing claim throws, so guard optional claims:
`has(user.claims.tier) && user.claims.tier == "pro"`.
</Info>

## `type: jwt`

Verifies arbitrary JWTs from any issuer against a JWKS endpoint, a shared secret
(HS\*), or a pinned SPKI public key. Provide **exactly one** of `jwksUrl`,
`secret`, or `publicKey`.

```sh
export OMNI_SECURITY_JSON='{
  "mode":"any",
  "providers":[{
    "type":"jwt",
    "jwksUrl":"https://auth.example.com/.well-known/jwks.json",
    "issuer":"https://auth.example.com",
    "audience":"omni-model",
    "algorithms":["RS256"]
  }]
}'
```

<ParamField path="header" type="string" default="authorization">
Header carrying the token.
</ParamField>
<ParamField path="scheme" type="bearer | none" default="bearer">
`bearer` strips a case-insensitive `Bearer ` prefix; `none` uses the raw,
trimmed header value.
</ParamField>
<ParamField path="jwksUrl" type="URL">
Remote JWKS endpoint for asymmetric keys.
</ParamField>
<ParamField path="secret" type="string">
Shared secret for HS\* algorithms.
</ParamField>
<ParamField path="publicKey" type="string">
SPKI PEM public key. Requires `algorithms` with **exactly one** entry.
</ParamField>
<ParamField path="algorithms" type="string[]">
Allowed JWS algorithms, e.g. `["RS256"]`.
</ParamField>
<ParamField path="issuer" type="string">
Expected `iss` claim. Unchecked when omitted.
</ParamField>
<ParamField path="audience" type="string | string[]">
Expected `aud` claim. Unchecked when omitted.
</ParamField>
<ParamField path="userIdClaim" type="string" default="sub">
Claim mapped to `user.id`.
</ParamField>
<ParamField path="deviceIdClaim" type="string">
Claim mapped to `device.id`, if present.
</ParamField>
<ParamField path="clockToleranceSeconds" type="integer" default="60">
Allowed clock skew, in seconds.
</ParamField>

<Warning>
Provide **exactly one** of `jwksUrl`, `secret`, or `publicKey` — the config
fails at startup otherwise. When you pin a `publicKey`, `algorithms` must list
exactly one algorithm.
</Warning>

## `type: firebase-auth`

Verifies Firebase Authentication ID tokens (RS256) against Google's
secure-token JWKS. The Firebase uid (`sub`) becomes `user.id`. Clients send
`Authorization: Bearer <ID token>`.

```sh
export OMNI_SECURITY_PROVIDERS_JSON='[
  {"type":"firebase-auth","projectId":"my-app-12345"}
]'
```

<ParamField path="projectId" type="string" required>
Firebase project id, e.g. `my-app-12345`. The verifier pins the issuer to
`https://securetoken.google.com/<projectId>` and the audience to `<projectId>`.
</ParamField>
<ParamField path="header" type="string" default="authorization">
Header carrying the ID token as `Bearer <token>`.
</ParamField>
<ParamField path="clockToleranceSeconds" type="integer" default="60">
Allowed clock skew, in seconds.
</ParamField>

## `type: supabase`

Verifies Supabase Auth access tokens, either with the project's legacy shared
JWT secret (HS256) or against the project's JWKS for asymmetric signing keys.
`sub` becomes `user.id`; Supabase claims like `role` and `app_metadata` are
available under `user.claims`.

```sh
export OMNI_SECURITY_PROVIDERS_JSON='[
  {"type":"supabase","url":"https://abcdefgh.supabase.co"}
]'
```

<ParamField path="url" type="URL">
Supabase project URL. Used to derive the JWKS endpoint
(`<url>/auth/v1/.well-known/jwks.json`) and issuer (`<url>/auth/v1`).
</ParamField>
<ParamField path="jwtSecret" type="string">
Legacy shared JWT secret (HS256).
</ParamField>
<ParamField path="jwksUrl" type="URL" default="<url>/auth/v1/.well-known/jwks.json">
Explicit JWKS endpoint for asymmetric keys.
</ParamField>
<ParamField path="issuer" type="string" default="<url>/auth/v1">
Expected issuer.
</ParamField>
<ParamField path="audience" type="string | string[]" default="authenticated">
Expected audience.
</ParamField>
<ParamField path="header" type="string" default="authorization">
Header carrying the access token as `Bearer <token>`.
</ParamField>
<ParamField path="clockToleranceSeconds" type="integer" default="60">
Allowed clock skew, in seconds.
</ParamField>

<Warning>
Provide `jwtSecret` **or** `jwksUrl`, not both. With neither, `url` is required
so the JWKS endpoint and issuer can be derived.
</Warning>

## Sending the token

All three verifiers read `Authorization: Bearer <token>` by default. Point any
OpenAI SDK at the proxy and pass the token as the API key.

<CodeGroup>
```bash cURL
curl https://your-proxy.example.com/v1/chat/completions \
  -H "Authorization: Bearer $ID_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4o-mini","messages":[{"role":"user","content":"hi"}]}'
```

```python Python
from openai import OpenAI

client = OpenAI(
    base_url="https://your-proxy.example.com/v1",
    api_key=id_token,  # sent as Authorization: Bearer <id_token>
)
client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "hi"}],
)
```
</CodeGroup>

An invalid or expired token is rejected with a token-free reason (for example
`token expired`, `token signature verification failed`) and an OpenAI-style
error body. The raw token is never logged or echoed.

## Combining verifiers

`security.mode` controls how multiple verifiers combine, and `publicPaths`
exempts specific routes from authentication.

<Tabs>
<Tab title="mode: any (default)">
Verifiers are consulted in config order. The first one that **accepts** wins;
a verifier whose credential is absent is skipped. Use this to accept tokens
from more than one issuer.

```sh
export OMNI_SECURITY_JSON='{
  "mode":"any",
  "providers":[
    {"type":"firebase-auth","projectId":"my-app-12345"},
    {"type":"supabase","url":"https://abcdefgh.supabase.co"}
  ]
}'
```
</Tab>
<Tab title="mode: all">
**Every** configured verifier must accept — for example, require a valid user
token *and* an app-attestation credential on the same request. Identities are
merged; the first defined `user.id`/`device.id` in config order wins.

```sh
export OMNI_SECURITY_JSON='{
  "mode":"all",
  "providers":[
    {"type":"firebase-auth","projectId":"my-app-12345"},
    {"type":"firebase-app-check","projectNumber":"1234567890"}
  ]
}'
```
</Tab>
</Tabs>

`publicPaths` bypasses authentication for exact paths, or by prefix when the
pattern ends with `*`:

```sh
OMNI__SECURITY__PUBLIC_PATHS='["/v1/models","/v1/models*"]'
```

<Note>
`/healthz` and verifier-contributed routes (such as `/auth/app-attest/*`) are
never behind authentication, regardless of `publicPaths`. With **no** providers
configured the proxy accepts unauthenticated `/v1/*` traffic and logs a warning
at startup.
</Note>
